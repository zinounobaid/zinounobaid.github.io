Overview

This project is a GPU-accelerated transient thermal simulation framework that models heat transfer in a 316L stainless steel immersion heater tank. The core physics problem: industrial heater tanks use a 1960s-era relay feedback loop with a single sensor, but the heater rods have significant thermal inertia — they continue dumping stored heat into the water after the relay opens, causing temperature overshoot. The simulation quantitatively proves that a 2-of-3 sensor voting system eliminates this overshoot.

The framework has been extended with a secondary module for EMI/EMC microwave leakage simulation through oven door gaskets, using the same voxelization engine and database architecture.

The Problem

Industrial heater tanks use a single-sensor relay feedback loop (1-of-1). The 316L stainless steel heater rods have significant thermal inertia — they continue dumping stored heat into the water after the relay opens, causing temperature overshoot at the exhaust that can damage downstream components or violate safety limits. The single sensor cannot detect this lag because it measures only one location.

The simulation quantitatively proves that a 2-of-3 voting system (exhaust probe, bulk fluid probe, rod surface probe) catches the overshoot earlier and makes better control decisions by accounting for the rod's stored thermal energy, reducing overshoot magnitude and duration.

Technical Architecture

1. GPU-Accelerated Physics Simulation (CUDA/CuPy)

  • Wrote a custom CUDA RawKernel implementing the Möller-Trumbore ray-triangle intersection algorithm on the GPU, performing point-in-mesh tests for voxelization at >40M points per batch — a 10–100x speedup over CPU trimesh.contains() fallback
  • Built a CUDA RawKernel that computes the full 3D transient energy balance (conduction + convection boundary conditions) in a single pass on the GPU, with harmonic mean thermal conductivity at material interfaces and Newton cooling at exterior boundaries
  • Implemented automatic GPU/CPU fallback: if CuPy is unavailable or VRAM is insufficient (<10% headroom), the solver falls back to NumPy with identical numerical behavior
  • Memory management: tiled batch processing to stay within VRAM budgets across RTX 2000 (16 GB) and RTX 3080 (10 GB) cards, with CuPy memory pool cleanup between batches

2. Computational Geometry & CAD Interoperability

  • Built a STEP file loader using OpenCascade (via CadQuery/OCP + trimesh) that extracts solid bodies from SolidWorks exports, auto-detects part numbers via regex pattern matching, and auto-assigns roles (heater, sensor, plug, fluid, hardware) based on known part number patterns
  • Implemented re-import diffing: SHA256 hashing detects CAD changes between revisions, and the loader compares body-by-body (unchanged/changed/added/removed), only requiring user input for new or modified parts
  • Voxelization with sub-grid bounding-box pre-filtering: query points are generated only within each body's bounding box (with 1% margin for numerical safety), avoiding OOM on coarse grids
  • Resolved microscopic CAD watertightness issues with a 1-voxel morphological dilation seal, then inverse 3D flood-fill from the grid boundary to identify internal cavities (the water domain)

3. Transient Thermal Solver (Node-Based FVM)

  • Implemented explicit Euler time integration solving the full energy balance per node: ρ·cp·V · dT/dt = Σ(k_eff · A/d · ΔT) for conduction between any two adjacent nodes across any material interface
  • Convection at solid-fluid interfaces with configurable heat transfer coefficient
  • Sparse heater source term applied to only ~0.1% of voxels (no full-array allocation)
  • Auto-computed stable timestep via Fourier criterion (dt = 0.45 · dx² / α_max)
  • Enhanced fluid effective thermal conductivity (k × 15) to model natural convection via Nusselt correlation
  • Harmonic mean conductivity at material interfaces ensures physically correct thermal resistance between dissimilar materials

4. Database Design (Versioned Simulation History)

  • Designed and implemented two SQLite databases: materials.db (shared library of materials with CLI manager) and projects.db (full versioned computation tree — Project → CAD Revision → Body Tags → Mesh → Simulation Run → Results)
  • Creates a "git for computations" where every revision, mesh, and run is permanently reviewable and comparable
  • Database stores compressed (zlib) numpy arrays for mask blobs, enabling immediate mesh loading without re-voxelization

5. Control System Modelling

  • relay_1of1(): Binary on/off hysteresis model of the existing 1960s relay loop (single sensor ± tolerance dead band)
  • voting_2of3(): Three sensors (exhaust, fluid, rod) each cast a vote; heater fires only if ≥2 agree
  • smart_inertia(): Enhanced 2/3 voting that checks rod temperature derivative (dT/dt) — if the rod is still heating the fluid from stored energy, it delays re-firing
  • Sensor thermal lag modelled as a two-stage RC low-pass filter (sleeve → potting compound → element), with time constants derived from physical geometry and material properties

6. 3D Visualization (PyVista)

  • Built an interactive 3D voxel viewer with GPU-accelerated point cloud generation (CuPy): for grids >200K voxels, coordinates are built entirely on the GPU and only visible (non-ambient) subsets are downloaded to CPU RAM
  • Material-colored rendering (blue = water, red = heater, green = hardware, yellow = sensor), semi-transparent CAD body overlay, sensor position markers
  • Keyboard controls: orthogonal slice planes, CAD toggle, liquid-only view, camera presets (X/Y/Z/iso views), legend toggle

7. PySide6 Desktop GUI

  • Modular panel-based architecture (material manager, DB manager, voxelizer, simulation runner, results viewer) with a tabbed dashboard
  • Real-time logging via QPlainTextEdit with auto-scroll, dynamic theme colors, and workflow-order navigation
  • Cross-modal interaction: clicking a mesh record in the DB manager opens the 3D voxel viewer; selecting a project shows its run history

8. EMI/EMC Microwave Leakage Simulation (Secondary Module)

  • Extended the framework to model microwave leakage through oven door gaskets, using the same voxelization engine and database architecture
  • Dedicated microwave solver module with standalone simulation panel, heatmap visualization dialog, and auto-recognition of door components (choke, gasket, cavity)
  • Addresses a real production issue: historically failed 2.45 GHz frequency on microwaves built at Dynamo

Key Problem-Solving Examples

The Water Domain Bug

The simulation produced NaN for fluid temperatures because the water fill algorithm assumed every Z-slice was a closed ring. The real CAD has inlet/outlet holes and heater rod penetrations that break the 2D boundary. The fix required shifting from a 2D slice-by-slice approach to a true 3D flood-fill with morphological sealing — a fundamentally different algorithm that took three iterations to get right.

The Voxel Bleed Bug

Material IDs were "bleeding" between adjacent bodies because the original voxelization applied bodies in arbitrary order. A heater rod that overlapped a fluid domain boundary would sometimes get overwritten by ambient ID. The fix was a priority-sorted approach: bodies are processed in ascending priority order (part=5 → sensor=8 → heater=10), so higher-priority materials always win at overlapping voxels.

The OOM Crisis

At 0.5mm resolution (70M voxels), generating all query points at once required ~3 GB of RAM just for coordinates. The fix was a bounding-box pre-filter: for each body, only generate points within its bounding box (typically <5% of the full grid). This made 0.5mm voxelization feasible on consumer GPUs.

Notes & Future Improvements

The project reached a complete end-to-end pipeline but has never produced a physically valid simulation run due to the water domain bug. Priority next steps include fixing the water fill algorithm (short-term: add solid plugs in SolidWorks to seal inlet/outlet holes; long-term: implement directed flood-fill from a known interior seed point), running the first valid end-to-end simulation, and cross-validating against physical test data from the production floor.

Future physics improvements include modelling heater thermal lag as two concentric bodies (inner element + outer sheath), adding advection terms for recovery time simulation under continuous flow draw, and running a mesh convergence study to confirm temperature fields stabilize.
